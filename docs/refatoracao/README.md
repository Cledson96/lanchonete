# Refatoração faseada — índice operacional

Este diretório contém o plano operacional em partes menores para executar a refatoração faseada do projeto com commit e push por fase.

## Documentos
- `../plano-refatoracao-faseada.md` — plano mestre
- `fase-0-preparacao.md` — baseline, contratos e estratégia mínima de testes
- `fase-0-baseline.md` — artefato executado da Fase 0
- `fase-1-dtos-serializacao.md` — DTOs compartilhados e normalização
- `fase-2-ajustes-arquiteturais.md` — rotas fora do padrão e helpers de resposta
- `fase-3-checkout.md` — modularização do checkout
- `fase-4-cardapio.md` — quebra do maior hotspot da UI
- `fase-5-whatsapp-service.md` — isolamento da integração externa
- `fase-6-workspaces-admin.md` — modularização dos workspaces administrativos
- `fase-7-consolidacao.md` — revisão final, cobertura e padronização

## Regra de execução
1. Ler o plano mestre.
2. Ler apenas a fase ativa.
3. Executar somente os itens da fase.
4. Validar a fase.
5. Commitar apenas os arquivos relevantes da fase.
6. Fazer push da branch.
7. Atualizar status antes de seguir para a próxima.
